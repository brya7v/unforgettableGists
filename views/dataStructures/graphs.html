Graphs

Vertex a vertex to a graph is what a node to a tree is.

Edge An edge is a connection between two nodes. This can have weights.

  • We can determine the most efficient ways to go from one vertex to another, we can have 2 fast links that are more efficient than 1 slow.
  • Bidirectional relationship represented as a line without arrows, we can go fron one vertex to another.
  • One way relationship edge has an arrow that indicates the direction.
  • Binary tree is a graph where the vertex have only two connections.

Adjacency matrix Table where we can see if vertex hast a connection to another or not.

- If theres connection it is represented with a 1 if not a 0.

- If the connections are weighted the connections are represented with the weight, and if theres no connection with a 0.

Adjacency list This is stored in an object, with key beign the vertex and the value being the vertex it has connection to.

- An adjacency list of a graph is the complete representation of the graph itself

- A has connection to E and B: { A: ["B", "E"] }

BigO

In the adjacency list

- Create a vertex O(1).

- Remove a vertex O(v + e).

- Create an edge O(1).

- Remove an edge O(e).


In the adjacency matrix

- Create a vertex O(v²).

- Remove a vertex O(v).

- Create an edge O(1).

- Remove an edge O(1).

Space complexity

In the adjacency list we only store existing connections O(v)

In the adjacency matrix we store existing connections and unexisting connections (wich makes it unneficient in space complexity O(v²))


As the In the adjacency matrix has to store all the unexisting connections, the adjacency is a lot more efficient in memory.

Constructor for a adjacency list.

  
            Class Graph{
                constructor(){
                    this.adjacencyList = {}
                }
            }
Add vertex

Add vertex method with no duplicates. Returns true if succes, return false if not.

  
            addVertex(vertex){
                if(!this.adjacencyList[vertex]){
                    this.adjacencyList[vertex] = []
                    return true
                }
                return false
            }
Remove vertex

Remove vertex method with no duplicates. Removes the edges first, then the vertex. Returns true if succes, return false if not.

Bidirectional advantage we just have to pop the connections of the vertex we want to delete, then we remove the vertex connection in each vertex poped.

  
            removeVertex(vertex){
                if(!this.adjacencyList[vertex]){

                    while(this.adjacencyList[vertex].length){
                        let temp = this.adjacencyList[vertex].pop()
                        this.removeEdge(vertex, temp)
                    }

                    delete this.adjacencyList[vertex]
                   
                    return this
                }
                return false
            }
Add edge

Add edge method with no duplicates. Adds edge to the vertex connections array. Returns true if succes, return false if not.

  
            addedge(vertex1, vertex2){
                if(this.adjacencyList[vertex1] && this.adjacencyList[vertex1]){

                    this.adjacencyList[vertex1].push(vertex2)
                    this.adjacencyList[vertex2].push(vertex1)

                    return true
                }
                return false
            }
Remove edge

remove edge method with no duplicates. removes edge to the vertex connections array. Returns true if succes, return false if not.

  
            removeedge(vertex1, vertex2){
                if(this.adjacencyList[vertex1] && this.adjacencyList[vertex1]){

                    this.adjacencyList[vertex1] = this.adjacencyList[vertex1].filter(v => v !== vertex2)
                    this.adjacencyList[vertex2] = this.adjacencyList[vertex2].filter(v => v !== vertex1)

                    return true
                }
                return false
            }
