React

React
  • Is a JavaScript library for building user interfaces.
  • Mobile apps and desktop apps fell very "reactive": things happen instantly, you don't wait for new pages to load or actions to start.
  • At first, websites didn't have this Smooth transitions behavior, traditionally in web apps, you click a link and wait for a new page to load. You click a button and wait for some action to complete.

  • With Javascript vanilla we can breack that slow behavior, with Js we can change what the user sees without fetching and reloading the page every time
  • Javascript runs in the browser on the Currently loaded page and it can manipulate the HTML structure (DOM) of the page.
  • Thus no visible request to the server are required, no need to wait for a new HTML page as a response and we can have a smooth behavior.

  • Thats why javascript can help us, and React is an extra building block that can help us with writing client side javascript code to have an even better reactive experience.
  • React its all about components
  • Every component has a clear task
  • React its not a step by step instructions code, in React we work at a higher level, creating components wich then create all those step by step instructions.
  • React
    • Is a client side JavaScript library
    • All about building modern, reactive user interfaces for the wab
    • It can help building complex applications easier
Alternative Frameworks

- Angular Complete component based UI framework, packed with features. Uses typescript. Can be an overkill for smaller projects.


- React is a lean and focused component based UI library. Certain features (ex. routing) area added via community packages.


- Vue Mixture of angular and react, is also a complete component based UI framework, includes most core features, less popular than react and angular.
