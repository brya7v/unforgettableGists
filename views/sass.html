Css
  • Cascading style sheets.
  • It can be coded inside an HTML tag (style attribute).
  • Css code can be inside an html document style element, inside the head element.
  • A Css external file can be imported in a <link href rel="stylesheet" type="text/css"> .
Css decalaration block #id { prop: value; }
Cascade in order importance > specificity > source order
  • Importance.
    • User !important declarations.
    • Author !important declarations.
    • Author decalarations.
    • User declarations.
    • Default browser declarations.

  • Specificity.
    • Inline styles.
    • Ids.
    • Classes, pseudoclasses, attributes.
    • User declarations.
    • Elements, pseudoelements.

  • Source.
    • Same specificity -> the last style declaration will remain.
* Universal selector (selects all elements). has no specificity value (0, 0, 0, 0).
BEM Block element modifier. Block__element—modifier defines structure for element classes, it helps when working with Sass.
Sass Syntactically Awesome Style Sheets.
  • Is a css preprocessor, an extension of css that adds power and elegance to the basic language.
  • It requires a comiler. Sass code -> Sass compiler -> compiled css code.
  • Sintax. 
                    Sass 						Scss

                .class						.class {
                    Background: red				    background: red;
                                                                }
  • Main Sass features.
    • Variables For reusable values such as colors, font-sizes, spacing, etc. Are declared with: $color-red-1: #d0d0d0
    • Nesting To nest selector inside of one another using BEM: 
      
        .class						
            background: black
            &-red
                color: red
    • Partials and imports To write Css in different fies and importing them all into one single file.
    • Mixins To write reusable pieces of css.
    • Functions Mimilar to mixins, with the difference tha they produce a value that can be used.
    • Extends To make different selectors inherit declarations that are common to all of them.
    • Control directives For writing complex code using condifionals and loops.
NPM Node Package Manager. Wee need NPM to install a Sass compiler. it helps when working with Sass.
  • Download and install npm.
  • npm init to initialize the package.json dependencies.
  • npm install node-sass --save-dev to install node-sass compiler and add it onyly to the dev dependencies inside the package.json.
  • Inside the package.json we can add a script to run the compiler.
    • 
      "scripts": {
        "compile:sass": "node-sass sass/main.sass css/style.css -w"  
      },
  • The files must be in the specified routes in order to work properly, the -w (--watch) matinans the script watching for changes on the files.

Responsive design
  • Adaptive content to different devices and resolutions.
  • Musts for a good responsive design.
    • Fluid layouts: A fluid layout expands items to fill available free space or shrinks them to prevent overflow.
      • the content adapts to the viewport and maintans its main usability/pourpose, we can achive this mainly in 4 different aproaches:
      1. Floating layouts
        • It consists in rows (class="row") with floating columns (class="col-1-of-3") inside.

          <div class="col-1-of-2 pink">1/2</div>
          <div class="col-1-of-2 pink">1/2</div>
        </div>

    • Media queries: Specific css style rules depending the resolution, two different aproaches(mobile first and desktop first) wich defines the media queries wr're going to use.
      1. Desktop first:***We write css for the desktop: large screens -> And then media queries for max width
        • @media only screen and (max-width: 1200px) 1023px) 767px) 480px)
      2. Mobile first: we write css for small screens -> and we write media queries for min-widths
        • @media only screen and (min-width: 480px) 767px) 1023px) 1200px)
    • Media queries breakpoints:
      • Bad: at specific resolutions of specific devices.
      • Good: multiple resolutions of multiple devices.
