Css - Sass

Css
  • Cascading style sheets.
  • It can be coded inside an HTML tag (style attribute).
  • Css code can be inside an html document style element, inside the head element.
  • A Css external file can be imported in a <link href rel="stylesheet" type="text/css"> .
Css decalaration block #id { prop: value; }
Cascade in order importance > specificity > source order
  • Importance.
    • User !important declarations.
    • Author !important declarations.
    • Author decalarations.
    • User declarations.
    • Default browser declarations.

  • Specificity.
    • Inline styles.
    • Ids.
    • Classes, pseudoclasses, attributes.
    • User declarations.
    • Elements, pseudoelements.

  • Source.
    • Same specificity -> the last style declaration will remain.
* Universal selector (selects all elements). has no specificity value (0, 0, 0, 0).
BEM Block element modifier. Block__element—modifier defines structure for element classes, it helps when working with Sass.
Sass Syntactically Awesome Style Sheets.
  • Is a css preprocessor, an extension of css that adds power and elegance to the basic language.
  • It requires a comiler. Sass code -> Sass compiler -> compiled css code.
  • Sintax. 
                    Sass 						Scss

                .class						.class {
                    Background: red				    background: red;
                                                                }
  • Main Sass features.
    • Variables For reusable values such as colors, font-sizes, spacing, etc. Are declared with: $color-red-1: #d0d0d0
    • Nesting To nest selector inside of one another using BEM: 
      
        .class						
            background: black
            &-red
                color: red
    • Partials and imports To write Css in different fies and importing them all into one single file.
    • Mixins To write reusable pieces of css.
    • Functions Mimilar to mixins, with the difference tha they produce a value that can be used.
    • Extends To make different selectors inherit declarations that are common to all of them.
    • Contorl directives For writing complex code using condifionals and loops.
Css selectors
  • Default: content-box -> the element size does not include padding neither border, its only defined by the content.
  • Better: border-box -> the element size includes padding and the outside of the border, this way it's easier working with sizes.

Responsive design
  • Adaptive content to different devices and resolutions.
  • Musts for a good responsive design.
    • Fluid layouts: A flex container expands items to fill available free space or shrinks them to prevent overflow.
      • the content adapts to the viewport and maintans its main usability/pourpose, we can achive this mainly in 3 different aproeaches:

    • Responsiveness across browsers: there are browser prefixes for specific styles, for the styles, in order to work on different browsers with different versions (caniuse.com)
    • Use of rem units and % instead of px. We can achive this with: 
      
        html
            font-size: 62.5%  // to set 1rem = 10px

        body
            font-size: 1.6rem // 16px
      And then modifying the font-size percentage in the media queries.

    • Flexible images: Relative units, different images for dif resolutions with the use of: 
      
        <picture>
            <source srcset="smoll.png 0.8x" media="(min-width:1000px)"> ( / 0.8 smaller (20% size inc))
            <source srcset="big.png 2x" media="(max-width:1000px)"> ( / 2x smaller (half size))
            <img>
        </picture>
